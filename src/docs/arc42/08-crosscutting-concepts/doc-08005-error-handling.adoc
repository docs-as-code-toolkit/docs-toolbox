[[error-handling]]
== Error Handling

The workflow shell steps use `set -euo pipefail` in the tag computation,
description extraction, and build/push steps. This means most command failures
fail the workflow instead of being silently ignored.

The README description extraction step has a fallback description if the marker
block is absent or empty. Docker command failures are surfaced through the
container process exit code.
